Compatibility issues with revisioned Anbernic RG XX models!

Please be aware that since the release of KNULLI Gladiator II, Anbernic has released and shipped revisioned models of some of their RG XX series devices. If you attempt to run KNULLI Gladiator II on a revisioned device, KNULLI will simply not boot.

Luckily, we are aware of the problem and we are currently running a public alpha test with builds that should be compatible with the new device models.

If you have problems installing KNULLI on a brand new RG XX device, either wait for the next public release of KNULLI or join us on Discord and type !revisions in the help channel to get access to the public alpha builds and help us test them.

    Extracting the kernel and bootloader¶

    This section is for developers

    This section is for developers who want to compile their own KNULLI build from source code. If you simply want to install KNULLI on your device, you can safely ignore this section and simply follow the Quick Start Guide.

    The KNULLI distribution for the RGCubeXX already includes a bootloader, u-boot, and kernel in binary form extracted from the stock firmware. There is no need to extract any of those manually unless you need them to compile KNULLI from scratch.

    Anbernic has not published the source code of the u-boot and kernel of the RGCubeXX. In order to have a working configuration you have two options:

    • You can use the stock kernel, bootloader, and u-boot from the stock SDCARD. See below for instructions on how to extract those.
    • You can compile the included kernel with the source code of this distribution, however, note that there will be a few elements that are missing

    The firwmare included in releases uses the kernel, bootloader, and u-boot from the stock firmware.

    You can read this page to learn how to extract the bootloader, u-boot, and kernel from the stock firmware.
